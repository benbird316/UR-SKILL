# 术语表

> 加载阶段：步骤 1（解析）
> 本文档定义 SKILL 中使用的自创专业术语，确保 body 中的术语含义明确无歧义。

---

## 术语定义

| 术语 | 英文对应 | 定义 |
|:---|:---|:---|
| 技术翻译工程 | Technical Translation Engineering | 本 SKILL 的核心能力域：将中文技术文档系统性地转换为严谨、准确的英文版本的全过程，涵盖术语查证、语境适配、质量评估、格式保持 |
| 非翻译区 | Non-Translatable Zone | 在技术文档翻译中不应被改动的区域，包括：代码块、行内代码、URL、占位符变量 |
| 术语映射表 | Terminology Map | 翻译过程中建立的中文术语到英文译法的对照表，每条记录包含置信度和来源 |
| 术语一致性 | Terminology Consistency | 同一中文技术术语在全文所有出现位置均使用同一英文译法 |
| 联网术语查证 | Web-Based Terminology Verification | 通过 WebSearch/WebFetch 工具在线搜索和确认技术术语的标准英文译法 |
| 格式保持 | Format Preservation | 翻译过程中保留原文档的 Markdown 结构、代码块、表格、公式等非文本元素不变 |
| 置信度 | Confidence Level | 对术语译法可靠性的三级评估：High（官方文档确认）、Medium（行业惯例确认）、Low（推断译法） |
| 风格得体性 | Style Appropriateness | 译文在语气、语域、句式复杂度上与目标文档类型（API文档/学术论文/README）的匹配程度 |
| 冗余套话 | Redundant Filler Words | 中文技术文档中常见的、在英译时可以省略的修饰性词语，如"进行""相关""实现" |
| 翻译说明 | Translation Notes | 在最终交付物中附加的、标注翻译过程中关键决策的说明文字 |
