# 方法类 ref：术语查证方法

> 加载阶段：步骤 2（调研）
> 回答"怎么找？"——技术术语英文译法的联网搜索策略、关键词模板和节流规则

---

## 1. 搜索优先级

按权威性从高到低排列，优先使用高优先级来源：

| 优先级 | 来源类型 | 示例 | 说明 |
|:---:|:---|:---|:---|
| 1 | 官方文档 | MDN Web Docs、Microsoft Learn、IEEE Xplore、各语言/框架官方文档 | 最权威，术语标准的来源 |
| 2 | 行业标准 | ISO、RFC、W3C、CNCF | 国际标准组织定义的术语 |
| 3 | 权威社区 | Stack Overflow ≥10 赞同、GitHub 官方 README/Discussions | 行业约定俗成的译法 |
| 4 | 技术百科 | Wikipedia、DevDocs | 参考性来源，需交叉验证 |

---

## 2. 搜索关键词模板

### 2.1 直接翻译搜索

适用于大多数通用技术术语：

```
"{中文术语}" English technical term
"{中文术语}" translation in "{领域}" documentation
```

示例：
- `"服务发现" English technical term`
- `"注册中心" translation in "microservices" documentation`

### 2.2 概念定义搜索

当直接搜索翻译找不到结果时，搜索术语的技术定义：

```
what is "{术语概念}" in {技术领域}
{术语概念} definition
```

示例：
- `what is "服务网格" in cloud native`
- `service mesh definition`

### 2.3 差异对比搜索

用于多义术语或易混淆术语：

```
"{中文术语}" "{相关术语}" difference
{英文A} vs {英文B}
```

示例：
- `"端口" "连接器" difference`
- `configuration vs settings`

### 2.4 站点限定搜索

针对特定权威源的精确搜索：

```
"{英文候选译法}" site:iso.org
"{英文候选译法}" site:ieee.org
"{英文候选译法}" site:kubernetes.io
```

## 3. 搜索节流规则

### 3.1 轮次限制

- 单次翻译任务最多 **4 轮** 联网搜索
- 每轮合并 **3-5 个** 待查术语批量搜索
- 优先合并同领域、同类型的术语

### 3.2 升级路径

```
第 1 轮：直接翻译搜索（通用模板）
   ↓ 未找到权威结果
第 2 轮：概念定义搜索 + 站点限定
   ↓ 仍未找到
第 3 轮：差异对比搜索 + 社区源交叉验证
   ↓ 仍未找到
第 4 轮：最后尝试，调整关键词或领域
   ↓ 全部失败
终止搜索 → 全部标注为 Low 置信度推断译法
```

### 3.3 降级触发条件

出现以下任一情况，该术语标注为推断译法（Low 置信度）：
- 4 轮搜索后仍无权威来源
- 仅在中文技术博客或非官方翻译中出现
- 多个来源译法不一致且无法判断权威性
- 属于中国特有技术概念，无标准英文对应

### 3.4 新兴术语处理

对于无法在权威源验证的新兴技术术语：
1. 基于构词规律和领域知识推断译法
2. 标注 `[Inferred: confidence low]`
3. 在 Notes 列说明推断依据
4. 建议用户根据实际情况确认
