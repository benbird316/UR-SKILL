# Anti-Patterns: Technical Document Lossless Semantic Optimization

> Used during static inspection to identify optimization behaviors that appear correct but are actually harmful.

## Anti-Pattern Checklist

| No. | Anti-Pattern | Detection Method | Fix Strategy |
|:---|:---|:---|:---|
| AP-01 | **Semantic Distortion**: Absolutizing probabilistic statements ("may lead to" -> "leads to"), converting possibility assertions into certainty | Scan for changes in fuzzy qualifiers: detect whether words like "may", "perhaps", "usually" were deleted or replaced before/after optimization | Restore original fuzzy qualifiers; mark as "Retainable Fuzzy Semantics", prohibit strengthening |
| AP-02 | **Information Loss**: Deleting modifiers that appear redundant but carry semantic constraints (e.g., risk warnings, conditional qualifiers, quantitative thresholds) | Baseline comparison: cross-reference the semantic baseline checklist, verify the retention status of each qualifier in the optimized document | Restore lost qualifiers and mark [Condition] or [Risk]; record restoration reason |
| AP-03 | **Causal Chain Breakage**: Compressing timelines or merging steps causing incoherent causal logic, mismatch between preconditions and post-conditions | Causal chain strength decay calculation: optimized confidence / baseline confidence <= 0.7 triggers alert | Trigger circuit breaker protection, rollback to form + vocabulary layer compression only; restore temporal structure |
| AP-04 | **Cross-Document Semantic Drift**: The same concept is optimized into different expressions across documents, breaking cross-document references | Terminology consistency check: scan different optimized versions of the same entity/concept across documents; verify conceptual definition consistency | Unify terminology; establish cross-document terminology mapping table; restore consistency |
| AP-05 | **Code Block Destruction**: Modifying code block content, deleting comment lines, or merging code lines during compression, causing syntax errors | Code block integrity check: verify code block count, language identifier, boundary markers; compare block-by-block with original document | Fully restore code blocks to original state; mark code blocks as indivisible units during optimization |
| AP-06 | **Numbering System Collapse**: Failing to synchronize numbering when adjusting document structure, causing cross-references to point to wrong sections/figures | Numbering consistency check: scan all cross-reference formats ("see Section X", "as shown in Figure X"), verify target exists and numbering matches | Globally renumber and verify all cross-references; mark all affected reference points |
| AP-07 | **Over-Compression**: Sacrificing readability for token compression rate, making the optimized document difficult to understand | Readability check: sentence length distribution, clause nesting depth, technical term density; compare with original document | Restore over-compressed paragraphs; ensure information density improvement is within the 30-50% reasonable range |
| AP-08 | **Structural Defect**: Deleting heading levels or altering the document skeleton during logical reorganization, causing structural collapse | Structural integrity check: compare heading hierarchy tree before/after optimization (H1->H2->H3), verify layer count and parent-child relationships are consistent | Restore heading hierarchy structure; maintain skeleton integrity during reorganization |
| AP-09 | **Ambiguous Reference**: Deleting operation objects or subjects during compression, making execution instructions unclear | Reference clarity check: scan all operation instructions, verify clear execution subject (who does it) and operation object (to what) | Restore full operation object names and mark [Object]; ensure all instruction objects are explicit |
| AP-10 | **Metadata Loss**: Removing document header metadata or footer traceability information during optimization | Metadata integrity check: verify that required metadata fields (attribution/scenario/version) are fully preserved | Restore complete metadata from original document; ensure version info and date format comply with specifications |
