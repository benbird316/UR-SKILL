# 方案类 ref 设计指南

> 只教怎么写方案类 ref 文件。方案类 ref 回答"怎么做？"——正反模式对照 + 修复策略 + 验证步骤。
> 判定是否需要生成方案类 ref，见 skill-package-design-guide.md §2。

---

## §1 格式模板

```markdown
> 加载阶段：步骤 4

## {问题类型}

**错误模式**：
```code
{错误代码/配置/操作}
```
- 问题：{为什么导致问题}
- 影响：{影响范围/严重性}

**正确模式**：
```code
{正确代码/配置/操作}
```
- 原理：{为什么是正确的}
- 注意事项：{应用时的边界条件}

**验证**：{修复后如何验证——具体到命令或检查项}
```

---

## §2 规则

- **MUST** 包含错误模式 + 正确模式的对照——只给正确模式不够
- **MUST** 包含影响的量化描述（影响哪个维度、什么程度），不做笼统标注
- **MUST** 验证步骤具体可执行（命令/检查项），不是"确认无误"
- **MUST NOT** 混入概念定义或检测方法（那是说明类/方法类的事）
- **SHOULD** ≤ 150 行。每个模式一般 15-25 行，4-6 个模式为宜。

---

## §3 反模式文件（三底线范畴，非方案类 ref）

> 注意：反模式文件（anti-patterns.md）属于**三底线**（SKILL 质量保障），不属于领域知识的方案类 ref。二者范畴不同。

反模式文件专门记录"看似正确实则有害"的做法，用于校验阶段扫描质量问题。

### 3.1 六个必要要素

| 要素 | 说明 | 为什么必须 |
|:---|:---|:---|
| **名称** | 简洁、精准的命名 | 便于引用和检索 |
| **表现** | 什么情况下出现，如何识别 | 让使用者能自行诊断 |
| **为什么出现** | 好意图（好心） | 反模式不是错误，是"好心办坏事" |
| **危害** | 为什么错（办坏事） | 让使用者理解后果 |
| **检测方法** | 如何自动化/手动识别 | 降低检查成本 |
| **规避策略** | 替代方案 + 为什么对 | 只说不做X不够，必须说"做Y instead" |

### 3.2 文件级结构

反模式文件分为两层：
1. **通用反模式引用**（固定）：声明通用反模式（占位符残留、规范越位、示例污染等）由 UR-SKILL 统一处理，生成 SKILL 时禁止重复罗列
2. **领域特有反模式**（按需）：仅列出该 SKILL 领域特有的反模式

### 3.3 单条反模式格式模板

```markdown
## {编号} {名称}

**表现**：{什么情况下出现，如何识别}

**为什么出现**：{好意图，开发者为什么会选择这个做法}

**危害**：{为什么错，具体后果}

**检测方法**：{如何自动化或手动识别}

**规避策略**：{替代方案} → {为什么对}
```

### 3.4 检测优先级

扫描顺序（高优先级 → 低优先级）：

| 优先级 | 反模式 | 理由 |
|:---|:---|:---|
| 极危 | 占位符残留 | 交付物不完整 |
| 高危 | 规范越位、风险边界滥用、步名不一致、IDE工具绑定、盲区甩锅、能力降级 | 直接影响安全/专业性 |
| 中危 | 架构混淆、切面填充、示例污染、检查缺失 | 直接影响质量深度 |
| 低危 | 盲区逃避 | 影响持续优化 |

### 3.5 ur-skill 自检清单

ur-skill 设计 SKILL 时，必须自检以下反模式：

| 反模式 | 表现 | 规避策略 |
|:---|:---|:---|
| 规范越位 | 将规范定义直接塞进 body | 删除规范表格，下沉到 references/ |
| 占位符残留 | body 中存在 [xxx]、TODO、FIXME 等 | 全部填充完整，或删除并标注盲区 |
| 示例污染 | 将 references/ 中的示例直接复制填充到 body | 示例通过声明式引用指向 references/ |
| 架构混淆 | 能力矩阵方向是工作流步骤 | 改为专业领域命名（如"静态分析"） |
| 检查缺失 | 审视检查数量不足 | 按节点类型分配：关键节点 6 维，非关键节点 3 维 |
| 盲区逃避 | 盲区识别只写"已标注局限" | 强制按三层机制执行 |

---

## §4 检查清单

- [ ] 每个模式包含错误模式 + 正确模式 + 验证
- [ ] 每个反模式包含六个要素（名称、表现、为什么出现、危害、检测、规避）
- [ ] 反模式有"为什么出现"（好意图）
- [ ] 反模式有"规避策略"（替代方案 + 为什么对）
- [ ] 反模式按检测优先级排序
- [ ] 文件 < 200 行，超过则拆分
- [ ] 无占位符残留（自身不能违反反模式）
