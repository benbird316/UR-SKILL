# 脚本文件设计指南

> 只教怎么写 scripts/ 下的可执行脚本。脚本由 LLM 生成，可能由自动化系统执行。
> 判定是否需要生成脚本，见 skill-package-design-guide.md §2。

---

## §1 安全规范

### 1.1 五个安全原则

| 原则 | 说明 |
|:---|:---|
| **只读优先** | 脚本优先读取，不修改文件系统 |
| **幂等性** | 多次执行结果一致 |
| **自包含** | 不依赖外部环境 |
| **最小权限** | 只请求必要的权限 |
| **输入验证** | 验证所有输入，拒绝非法输入 |

### 1.2 禁止的操作

**MUST NOT** 执行文件系统修改（删除、写入、重命名）。
**MUST NOT** 执行网络请求（HTTP/HTTPS、API 调用、下载）。
**MUST NOT** 执行系统命令（shell 命令、子进程、系统调用）。
**MUST NOT** 修改环境变量（PATH、系统配置）。
**MUST NOT** 访问敏感信息（密码、密钥、Token、配置文件）。

### 1.3 允许的操作

- 读取文件（只读访问，不修改）
- 正则匹配（文本扫描、模式识别）
- 字符串处理（分割、拼接、格式化）
- 数学计算（数值运算、统计）
- 数据结构操作（列表、字典、集合操作）

## §3 脚本类型

| 类型 | 用途 | 安全等级 | 示例 |
|:---|:---|:---:|:---|
| **扫描脚本** | 静态检查, 识别问题 | 安全 | 反模式扫描、占位符检测 |
| **验证脚本** | 验证格式、约束、合规性 | 安全 | YAML frontmatter 验证 |
| **生成脚本** | 生成内容、填充模板 | 中风险 | 批量生成 SKILL 文件 |
| **转换脚本** | 格式转换、数据迁移 | 中风险 | Markdown 转 YAML |
| **部署脚本** | 部署到生产环境 | 高风险 | 自动部署、CI/CD 流水线 |

> 中风险及以上脚本需人工审核, 不能自动执行。

## §4 YAML 配置文件中的正则书写规范

当脚本通过 YAML 加载正则表达式时, 必须遵守以下规范, 避免转义错误和运行时 `re.PatternError`。

| 场景 | 规范 | 示例(目标正则: `\[`) | 反例(会导致双反斜杠) |
|:---|:---|:---|:---|
| 单引号字符串 | 使用 `\\` 表示一个反斜杠 | `'\\['` | `'\\\\['` |
| 双引号字符串 | 使用 `\\` 表示一个反斜杠 | `"\\["` | `"\\\"` |
| 验证方法 | 加载后立即 `re.compile(pattern)` 验证 | `re.compile(config['pat'])` | 仅在运行时 `findall` 才编译 |
| 可读性优先 | 复杂正则优先放入脚本常量(raw string), 不放入 YAML | `r"^\d+$"` | 在 YAML 中写 `'^\\d+$'` |

## §5 检查清单

- [ ] 脚本有安全等级标注(安全/中风险/高风险)
- [ ] 脚本有副作用说明(无/读取/写入/网络)
- [ ] 脚本有输入验证函数
- [ ] 脚本不执行高危操作(删除、写入、网络、系统命令)
- [ ] 脚本输出到 stdout, 不直接写入文件
- [ ] 脚本有错误处理(try/except)
- [ ] 脚本有日志记录(操作日志, 便于审计)
- [ ] 中风险及以上脚本经过人工审核
- [ ] 脚本自包含, 不依赖外部环境
- [ ] 脚本幂等, 多次执行结果一致
- [ ] 文件 < 200 行, 超过则拆分
