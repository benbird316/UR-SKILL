# 边界设计指南

> 只教怎么写 SKILL 的风险边界与专业边界。风险边界是"不做有害的事"，专业边界是"不做超出专业边界的事"，两者都不是能力降级。
> 判定边界设计需求，见 skill-package-design-guide.md §2。

---

## §1 三概念定义

### 1.1 风险边界（安全红线）

| 维度 | 内容 |
|:---|:---|
| **本质** | "不做有害的事"——安全底线 |
| **性质** | 安全性质，触及即终止任务 |
| **数量** | 由领域安全需求决定，通常 3-5 条 |
| **判定** | 触及后 SKILL 本身失去存在意义 |

**正例**：

| 编号 | 声明 | 理由 |
|:---|:---|:---|
| 01 | 不生成可执行恶意代码（勒索软件、挖矿脚本） | 触及 = SKILL 在助纣为虐 |
| 02 | 不基于种族/性别/年龄歧视性地标记或处理数据 | 触及 = SKILL 主动制造歧视 |
| 03 | 不输出违法内容或侵犯隐私的信息 | 触及 = SKILL 违法 |

**反例（能力降级伪装）**：

| 滥用写法 | 问题 |
|:---|:---|
| 只负责安全审查，不检查代码错误 | 能力降级——安全检查中发现的代码错误也应报告 |
| 只做代码风格检查，不分析功能正确性 | 能力降级——功能错误是风格检查的自然延伸 |
| 性能问题不在检查范围内 | 能力降级——声明"不做X"，但X是任务的自然部分 |

### 1.2 专业边界（越界防护）

| 维度 | 内容 |
|:---|:---|
| **本质** | "不做超出专业边界的事"——不越界替代专业角色 |
| **性质** | 范围性质，触及即终止越界行为，明确告知用户 |
| **数量** | 由领域范围决定，通常 1-3 条 |
| **判定** | 防止 SKILL 越界进入需要专业资质的领域 |

**正例**：

| 编号 | 声明 | 适用领域 | 理由 |
|:---|:---|:---|:---|
| 01 | 仅提供病情分析参考，不可替代医生开具处方或诊断 | 医疗 | AI 无权行使执业医师的处方权 |
| 02 | 输出为风险评估参考，不包含具体买卖时点与价位 | 金融 | AI 无权给出操盘级投资指令 |
| 03 | 仅提供法律知识解释，不构成正式法律意见 | 法律 | AI 无权出具具有法律效力的意见书 |

### 1.3 能力降级（反模式）

| 维度 | 内容 |
|:---|:---|
| **本质** | "做X但选择不报Y"——Y是X的自然延伸，但SKILL主动拒绝 |
| **性质** | 推卸责任，不是边界 |
| **判定** | SKILL 已有能力但主动拒绝，属推卸责任 |

**典型模式**：
- "只做X不做Y" —— 但Y是X的自然延伸
- "X不在检查范围内" —— 但X是当前能力的合理预期
- "不负责X" —— 但X是当前任务的合理组成部分

## §2 三概念速查表

| | 风险边界（安全红线） | 专业边界（越界防护） | 能力降级（反模式） |
|:---|:---|:---|:---|
| **本质** | 不做有害的事 | 不做超出专业边界的事 | 做X但选择不报Y |
| **性质** | 安全性质，触及即终止 | 范围性质，触及即终止越界行为 | 推卸责任 |
| **位置** | 风险边界声明 | 专业边界声明 | 不应存在 |
| **触及后果** | 立即终止任务 | 终止越界行为 + 告知用户 | 不应出现 |
| **判定钥匙** | "不做有害的事" | "不越专业边界" | "做X但不报Y，Y是X自然延伸" |

## §3 常见陷阱

### 3.1 "能力降级伪装成专业边界"

```markdown
专业边界-01：只做代码审查，不检查性能问题
```

**问题**：代码审查中发现的明显性能问题（O(n²)循环、内存泄漏）是代码质量的自然延伸，不应回避。

**修正**：要么删除（代码审查包含性能问题的基本识别），要么改为真正的专业边界：
```markdown
专业边界-01：仅提供代码层面的建议，不涉及生产环境部署方案的制定
```

### 3.2 "能力降级伪装成风险边界"

```markdown
风险边界-01：不负责用户输入的安全性验证
```

**问题**：如果任务是代码审查，输入验证问题就是审查内容的一部分。用风险边界免除责任 = 能力降级。

**修正**：安全相关的真正风险边界：
```markdown
风险边界-01：不生成可绕过安全机制的代码（如禁用CSRF验证、明文存储密码）
```

### 3.3 "把专业边界写成风险边界"

```markdown
风险边界-01：不提供具体买卖时点与价位
```

**问题**：这是专业越界防护（金融顾问权限），不是安全红线。应放入专业边界声明。

### 3.4 "用 MUST NOT 重复声明内容"

```markdown
MUST NOT 生成恶意代码
MUST NOT 基于种族歧视
MUST NOT 注入恶意提示
```

**问题**：这些已经在风险边界声明中列出，MUST NOT 重复是冗余。

**修正**：MUST NOT 引用声明：
```markdown
MUST NOT 违反风险边界声明中的任一条安全红线
```

## §4 检查清单

### 风险边界
- [ ] 每条风险边界都是安全红线，触及后 SKILL 失去存在意义
- [ ] 不含能力降级措辞（"不负责""只做X不做Y""不在范围内"）
- [ ] 不含专业边界内容（"不给处方""不给价位"）
- [ ] 是该领域特有的安全风险，非通用声明
- [ ] 数量由领域安全需求决定，不设硬性上限

### 专业边界
- [ ] 每条专业边界都是越界防护，防止替代专业角色
- [ ] 不含能力降级措辞（"只做X不做Y"而Y是X的自然延伸）
- [ ] 不含安全红线内容
- [ ] 是对该领域高度专业化边界的合理限制
- [ ] 数量由领域范围决定，通常 1-3 条

### MUST NOT
- [ ] 包含两条引用边界声明的基准规则
- [ ] 不重复边界声明的具体内容
- [ ] 领域特有的 MUST NOT（如存在）控制在 1-3 条
