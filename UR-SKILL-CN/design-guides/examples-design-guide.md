# 示例文件设计指南

> 只教怎么写 references/examples.md。示例是少样本学习的核心，必须解释"为什么是这个输出"。
> 判定是否需要生成示例文件，见 skill-package-design-guide.md §2。

---

## §1 标准格式

### 1.1 五个必要要素

| 要素 | 说明 | 为什么必须 |
|:---|:---|:---|
| **场景** | 什么情况下使用这个示例 | 让 LLM 理解适用范围 |
| **Input** | 用户输入 | 定义输入格式和内容 |
| **Output** | 期望输出 | 定义输出格式和内容 |
| **Rationale** | 为什么是这个输出 | 教学核心；没有 Rationale，示例变成简单映射 |
| **边界说明** | 为什么不是其他输出 | 让 LLM 理解什么情况下不能产生某个输出 |

### 1.2 格式模板

```markdown
## 示例1: {场景描述}

**Input**:
```
{用户输入内容}
```

**Output**:
```
{期望输出内容}
```

**Rationale**: {为什么是这个输出，核心判断逻辑}

**边界说明**: {什么情况下不能产生这个输出，常见错误}
```

### 1.3 三种示例类型

| 类型 | 说明 | 要求 |
|:---|:---|:---|
| **典型场景** | 基础用例，展示核心行为 | 所有 SKILL 必填至少 1 个 |
| **边界场景** | 边缘条件处理，展示行为切换点 | 输出格式必须与典型示例完全一致，不得引入新结构元素 |
| **错误场景** | 错误输入的处理，展示拒绝或修复逻辑 | 功能型 SKILL 必填，创意型推荐 |

---

## §2 示例数量

示例数量由 research-analyst 调查分析结果推导，无固定数值。

> 质量比数量更重要：1 个高质量示例 > 3 个低质量示例。

---

## §3 检查清单

- [ ] 示例数量由 research-analyst 调查分析结果推导
- [ ] 每个示例包含五个要素（场景、Input、Output、Rationale、边界说明）
- [ ] 每个示例有 Rationale（为什么是这个输出）
- [ ] 边界/错误示例有边界说明（为什么不是其他输出）
- [ ] 示例 Output 完整无占位符
- [ ] 示例与 body 不重复（示例在 references/，body 只引用）
- [ ] 文件 < 200 行，超过则拆分
