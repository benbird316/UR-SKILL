# 工具调用设计指南

> 只教怎么将抽象"动作"映射为具体工具调用。动作是"做什么"，工具是"用什么做"。
> 判定工具绑定需求，见 skill-package-design-guide.md §2。

---

## §1 跨 IDE 通用工具类别

生成的 SKILL.md 默认使用"通用类别"。若可确定目标平台，按 [skill-package-design-guide.md §A.2](skill-package-design-guide.md) 映射表替换为平台特定工具名。UR-SKILL 自身文件（SKILL.md、agent/*.md）**MUST** 始终保持通用类别（跨平台源码）。

| 通用类别 | 功能 | 输入要求 |
|:---|:---|:---|
| `[文件读取]` | 读取已知路径的文件内容 | 绝对路径 |
| `[文件写入]` | 创建或覆盖文件 | 路径 + 内容 |
| `[文件编辑]` | 精确替换文件内容 | 路径 + old_str + new_str |
| `[文件名搜索]` | 按 glob 模式查找文件 | glob 模式 |
| `[文本搜索]` | 按正则搜索文件内容 | 正则 + 路径 |
| `[语义搜索]` | 按意图/概念搜索代码 | 自然语言问题 |
| `[目录浏览]` | 列出目录结构 | 绝对路径 |
| `[联网搜索]` | 搜索网络实时信息 | 搜索关键词 |
| `[联网抓取]` | 获取指定 URL 内容 | URL |
| `[命令执行]` | 运行 Shell 命令 | 命令字符串 |
| `[诊断信息]` | 获取 IDE 诊断/lint 结果 | 文件 URI（可选） |

**语义自明的工作流原语**（无需转换）：

| 原语 | 功能 |
|:---|:---|
| `[Task]` | 派生子代理独立执行复杂子任务 |
| `[AskUserQuestion]` | 中断执行，向用户提问 |
| `[Skill]` | 调用另一个专项 SKILL |

语义自明，无需转换。生成 SKILL 时若可确定目标平台且有对应的子Agent/子SKILL 机制，可使用平台原生调用方式；否则保持原语形式。

## §2 选型决策矩阵

| 用户需求 | 推荐类别 | 次选类别 | 不推荐 |
|:---|:---|:---|:---|
| 搜索某个类的实现 | `[语义搜索]` | `[文本搜索]` | `[文件名搜索]` |
| 查找所有 `.ts` 文件 | `[文件名搜索]` | `[目录浏览]` | `[文本搜索]` |
| 阅读已知文件 | `[文件读取]` | — | `[文本搜索]` / `[命令执行] cat` |
| 搜索字符串 "todo" | `[文本搜索]` | — | `[语义搜索]` |
| 了解代码库架构 | `[语义搜索]` | `[文件名搜索]` + `[文件读取]` | `[文本搜索]` |
| 获取外部最新信息 | `[联网搜索]` | `[联网抓取]` | — |
| 运行测试/构建/lint | `[命令执行]` | — | — |
| 获取 git diff | `[命令执行]` (`git diff`) | — | — |
| 修改代码 | `[文件编辑]` | `[文件写入]` | — |
| 列出项目结构 | `[目录浏览]` | `[文件名搜索]` | `[命令执行] ls` |

## §3 动作-工具映射规范

### 3.1 标准映射格式

```
动作:
1. [文件读取] 读取用户需求描述 → 提取任务类型与目标系统
2. [命令执行] git diff origin/HEAD... → 获取代码变更清单
3. [文件名搜索] **/*.py 枚举变更文件 → 确定审查范围
4. [语义搜索] "where is input validation located?" → 定位现有校验器
```

**错误示例**（仅有抽象描述）：
```
动作:
1. 读取用户需求，提取任务类型
2. 获取代码变更清单
3. 确定审查范围
4. 定位现有校验器
```

### 3.2 参数化规范

| 动作描述 | 工具类别 | 示例参数 |
|:---|:---|:---|
| 获取当前分支变更 | `[命令执行]` | `git diff --name-only origin/HEAD...` |
| 搜索注入漏洞模式 | `[文本搜索]` | `pattern:"execute\(|eval\(|exec\(" glob:"*.py"` |
| 联网调查行业标准 | `[联网搜索]` | `query:"OWASP Top 10 2026"` |
| 理解模块职责 | `[语义搜索]` | `"What is the responsibility of AuthService?"` |

### 3.3 组合调用模式

```
动作:
1. 确定审查范围:
   a. [命令执行] git diff --merge-base origin/HEAD → 获取 diff
   b. [命令执行] git diff --name-only origin/HEAD... → 枚举变更文件
   c. 若 a 失败 → [命令执行] git diff HEAD~1（降级方案）
2. 审查每个变更文件:
   a. [文件读取] 读取文件完整内容 → 识别变更上下文
   b. [语义搜索] "what validators exist in this project?" → 获取项目校验模式
```

## §4 降级与容错设计

| 工具 | 首选 | 降级1 | 降级2 | 最后手段 |
|:---|:---|:---|:---|:---|
| 获取 diff | `git diff --merge-base origin/HEAD` | `git diff origin/HEAD...` | `git diff HEAD~1` | `AskUserQuestion` 请求指定范围 |
| 代码搜索 | `[语义搜索]` | `[文本搜索]` | `[文件名搜索]` + `[文件读取]` | 声明信息边界 |
| 联网调研 | `[联网搜索]` | `[联网抓取]` | 声明知识截止日期 | 请求用户补充 |
| 文件读取 | `[文件读取]` | — | — | 声明无法访问 |

**不可降级的底线操作**（失败必须终止当前步骤）：
- `[命令执行]` 执行 git 操作获取 diff
- `[文件读取]` 读取代码文件

## §5 按工作流节点类型的工具绑定

| 节点类型 | 核心工具类别 | 示例 |
|:---|:---|:---|
| **解析节点** | `[文件读取]`、`AskUserQuestion` | `[文件读取] 读取用户需求输入 → 提取任务类型` |
| **调研节点** | `[联网搜索]`、`[语义搜索]`、`[文件名搜索]` | `[联网搜索] query="{关键词}" → 获取行业标准` |
| **执行节点** | `[命令执行]`、`[文件读取]`、`[语义搜索]`、`[文本搜索]` | `[命令执行] git diff --merge-base origin/HEAD → 获取变更` |
| **校验节点** | `[文本搜索]`（占位符扫描）、`[文件读取]`（交叉验证） | `[文本搜索] pattern="\{.*\}" 扫描输出 → 检查占位符残留` |
| **交付节点** | `[文件写入]`、`[文件编辑]` | `[文件写入] 生成最终 SKILL.md → 路径 {output_path}` |

## §6 声明策略

声明策略由 research-analyst 根据 SKILL 的实际工具调用密度推导，无固定模板：

- 工具调用少的 SKILL：内联声明（动作中直接写 `[工具类别]`）
- 工具调用多的 SKILL：内联 + 集中式（末尾添加工具参考表）
- 有 scripts/ + assets/ 的 SKILL：集中式为主，内联为辅

---

## §7 反模式

| 编号 | 反模式 | 表现 | 修正 |
|:---:|:---|:---|:---|
| 1 | 无工具绑定 | 动作只写"读取代码"，不指定用 `[文件读取]` 还是 `[命令执行] cat` | 每个动作必须以 `[工具类别]` 开头 |
| 2 | 工具错配 | 用 `[联网搜索]` 搜索本地代码，用 `[文本搜索]` 做语义理解 | 参照 §2 选型矩阵 |
| 3 | 无降级路径 | 工具调用无 fallback，单点故障导致整个步骤失败 | 为关键工具调用提供至少 1 个降级路径 |
| 4 | 混用专用工具与 Shell | 动作中同时出现 `[文件读取]` 和 `cat`、`grep` | 一律使用专用工具类别 |
| 5 | 参数空洞 | 写 `[联网搜索] query:"搜索相关内容"` 无具体关键词 | 参数必须具体 |
| 6 | 未识别平台时绑定特定工具名 | 不确定目标平台的情况下在 SKILL.md 中写 `Read`、`RunCommand` 等 | 默认用通用类别；确定平台后按映射表替换（见 skill-package §A.2） |

---

## §8 检查清单

- [ ] 每个工作流步骤的动作都包含至少 1 个工具类别调用
- [ ] 工具调用使用通用类别格式（如 `[文件读取]`），**无** 特定 IDE 工具名
- [ ] 工具调用符合选型矩阵（§2），无跨类别混用
- [ ] 关键工具调用有降级路径
- [ ] 无 Shell 命令替代专用工具类别
- [ ] 参数具体而非空洞占位符
- [ ] 工具调用密度高的 SKILL 包含集中式工具参考表
- [ ] 不可降级的底线操作已标记（失败即终止）
- [ ] `AskUserQuestion` 作为最后手段时，提供了具体选项
