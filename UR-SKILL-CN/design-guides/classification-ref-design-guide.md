# 说明类 ref 设计指南

> 只教怎么写说明类 ref 文件。说明类 ref 回答"这是什么？"——领域分类体系 + 每类特征 + 判定标准。

## 格式

```markdown
> 加载阶段：步骤 1

## {分类名称}

**定义**：{一句话定义}

**特征**：
- {可观测的判定特征 1}
- {可观测的判定特征 2}

**判定标准**：{什么条件下判定为此类}

**优先级**：{该分类的严重程度/处理顺序，如 P0/P1/P2}

**常见误判**：
- "看起来像 X 其实是 Y" 的典型歧义案例
- 边界灰色地带如何判定

**相关标准/来源**：{外部知识来源及获取日期，如 K1 — OWASP 2024}
```

## 规则

- 必须列出**可观测**的判定特征——不是描述，是能做二元判断的具体指标
- 分类必须互斥 + 穷尽（MECE）
- 不写检测方法（那是方法类的事）、不写修复策略（那是方案类的事）
- 每种分类后若有必要跟子分类，同样遵循 MECE
- 必须标注**常见误判**——说明类和它的"近亲"之间的歧义边界
- 必须标注**相关标准/来源**（K类 + 信源 + 获取日期），遵循 ref-types-design-guide.md §1.2 时效性原则

## 长度

≤ 150 行。分析阶段知识通常最轻量。超过则检查是否混入了方法论或修复内容。
