# 能力架构设计指南

> 用途：说明如何设计能力矩阵、能力切面、边界声明。回答"为什么这么设计"和"怎么判断设计是否正确"。
> 核心原则：能力矩阵是能力域，不是工作流步骤；能力切面只针对核心领域；边界声明分为安全红线与越界防护。

---

## 1. 能力矩阵是能力域，不是工作流步骤

> 核心原理见 [design-rationale/design-rationale.md §1](../design-rationale/design-rationale.md#L8)

### 1.1 关键区分方法

使用排序测试和三问筛选判断候选域是能力域还是工作流步骤。

**排序测试**：将候选辐射领域重新排列，若逻辑崩坏，则是工作流步骤。

**三问筛选**：对每个候选领域问：
1. **独立性**：能否脱离其它领域单独定义？
2. **不可替代性**：缺少它是否会导致核心任务无法完成？
3. **互补性**：它与其他领域是并行协作关系，而不是先后依赖关系？

**反例**：候选域 A 解析需求 → B 检索资料 → C 生成报告，重排后逻辑崩坏 → 是工作流步骤。
**正例**：A 静态分析 / B 安全审计 / C 性能诊断，重排后仍然成立 → 是能力域。

### 1.2 领域数量由任务分析确定

> 核心原理与完整确定方法见 [design-rationale/design-rationale.md §2](../design-rationale/design-rationale.md#L16)

辐射领域数量推荐 **3-8 个**，具体数量由任务分析确定。使用排序测试和三问筛选确保每个辐射领域是独立且互补的能力域，而非工作流步骤。

---

## 2. 能力矩阵深度固定 4 层

> 核心原理见 [design-rationale/design-rationale.md §3](../design-rationale/design-rationale.md#L32)

每个领域固定 **4 层深度**（基础层、进阶层、高阶层、拓展层），不绑定 SKILL 复杂度。填写时确保每一层都有该领域对应能力水平的具体描述。

---

## 3. 能力切面只针对核心领域

能力切面刻画核心领域需要具备的能力维度，不是每个辐射领域都重复建设。

| 切面 | 核心问题 |
|:---|:---|
| 效率成本 | 资源投入是否合理？ |
| 知识深耕 | 知识覆盖是否完整？ |
| 风险识别 | 有什么潜在风险？ |
| 质量检验 | 输出质量是否可靠？ |
| 领域融合 | 领域之间是否协同？ |
| 系统全局 | 是否考虑了全局影响？ |

**任务锚定要求**：每个切面必须包含该核心领域的具体知识，不能只写"掌握相关领域知识""识别潜在风险"等通用套话。

**验证方法**：把切面内容中的任务名遮住，还能判断出是哪个任务 → 不合格；遮住后判断不了 → 合格。

---

## 4. 三层关系

| 层级 | 形式 | 回答的问题 |
|:---|:---|:---|
| 能力矩阵 | 矩阵（核心领域 + 辐射领域 × 4 层深度） | "有什么能力""能力有多深" |
| 能力切面 | 列表（6 个切面，只针对核心领域） | "核心领域需要具备哪些能力维度" |
| 工作流 | 纵向递进（步骤 → 动作 → 检查清单） | "怎么执行" |

三者互不替代。如果能力切面与能力矩阵重叠，说明能力切面设计错误；如果能力矩阵与工作流步骤重叠，说明能力矩阵设计错误（反模式4 架构混淆）。

---

## 5. 边界声明

边界声明分为风险边界（安全红线）与专业边界（越界防护）。详细设计方法见 [design-guides/boundary-design-guide.md](../design-guides/boundary-design-guide.md)。

### 5.1 风险边界

- 本质："不做有害的事"
- 性质：安全性质，触及即终止任务
- 数量：由领域安全需求决定，通常 3-5 条

### 5.2 专业边界

- 本质："不做超出专业边界的事"
- 性质：范围性质，触及即终止越界行为并告知用户
- 数量：由领域范围决定，通常 1-3 条

---

## 6. 常见错误

| 错误 | 问题 | 修正 |
|:---|:---|:---|
| 把工作流步骤伪装成能力域 | 能力矩阵变成执行顺序 | 执行排序测试和三问筛选，改为专业领域命名 |
| 为每个辐射领域都写 6 个切面 | 信息爆炸，注意力稀释 | 能力切面只针对核心领域 |
| 风险边界写成能力降级 | 安全刹车变免责声明 | 只声明安全红线，不做X不做Y放入能力范围描述或删除 |
| 切面填充通用套话 | 失去任务锚定 | 每个切面包含具体工具名/规范名/漏洞类型 |

---

## 7. 检查清单

- [ ] 能力矩阵覆盖全部核心需求
- [ ] 辐射领域通过排序测试和三问筛选
- [ ] 能力矩阵不是工作流步骤的别名
- [ ] 每个领域 4 层完整
- [ ] 能力切面 6 个，只针对核心领域
- [ ] 每个切面任务锚定，不含通用套话
- [ ] 风险边界是安全红线，不含能力降级
- [ ] 专业边界是越界防护，不含能力降级
