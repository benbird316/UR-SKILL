# SKILL 模板

> 用途：根据生成的 SKILL 类型（功能型/创意型/社交型）选择输出详略。所有 SKILL 共享 11 步工作流框架，子 Agent 调用按文件依赖判定可跳过。
> 核心原则：输出规格必须包含格式类型、输出结构、问题分级、判决策略、用户交互模式
> 设计方法详见 [design-guides/output-design-guide.md](../design-guides/output-design-guide.md)

## 4. 输出约束（通用）

- body 必须 < 500 行
- YAML frontmatter 必须包含 3 字段：name, description, metadata.updated
- description 是**唯一触发字段**，必须含中文触发短语
- 不得将 references/ 内容直接填充到 body 中
- 目录名使用复数形式：references/、scripts/、assets/
- 门控节点（调研、规划、校验、验证、循环判定）6 维全开
- 执行节点（解析、协调、分发、汇总、执行、组装）3 维
- 能力矩阵：核心领域 1 个，辐射领域数量由任务分析确定（推荐 3-8 个）
- 能力矩阵：每个领域 4 层深度（基础 → 进阶 → 高阶 → 拓展）
- 能力切面：只针对核心领域，6 个切面
- 风险边界声明：安全红线（通常 3-5 条）
- 专业边界声明：越界防护（通常 1-3 条）
- 审查/测试类 SKILL：输出规范必须包含格式类型、强制可视化、问题分级（极危/高危/中危/低危）、判决策略、用户交互模式
- 强制可视化的 SKILL：Mermaid 图表颜色必须显式指定，禁止依赖主题默认色
- 审查类 SKILL：输出结构必须包含执行摘要 + 正面观察 + 推荐行动
