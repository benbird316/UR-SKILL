# 示例文件模板

> 设计方法详见 [design-guides/examples-design-guide.md](../design-guides/examples-design-guide.md)

---

## 示例1：{典型场景}

**Input**：
```
{用户输入内容}
```

**Output**：
```
{期望输出内容，完整无占位符}
```

**Rationale**：{为什么是这个输出，核心判断逻辑}

**边界说明**：{什么情况下不能产生这个输出，常见错误}

---

## 示例2：{边界场景}

**Input**：
```
{边界条件输入}
```

**Output**：
```
{边界条件下的期望输出}
```

**Rationale**：{为什么是这个输出，边界处理逻辑}

**边界说明**：{什么情况下会走向另一个分支}

---

## 示例3：{错误场景}（复杂复杂度必填）

**Input**：
```
{错误输入}
```

**Output**：
```
{错误处理输出，明确拒绝或给出修复建议}
```

**Rationale**：{为什么不能按字面执行，潜在风险是什么}

**边界说明**：{与正确示例的关键区分点}

---

## 完整性检查清单

- [ ] 示例数量与复杂度匹配（简单 1-2，中等 2-3，复杂 3+）
- [ ] 每个示例包含五个要素：场景、Input、Output、Rationale、边界说明
- [ ] Output 完整无占位符
- [ ] 示例与 body 不重复
- [ ] 文件 < 200 行
