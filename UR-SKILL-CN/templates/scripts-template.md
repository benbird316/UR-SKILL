# 脚本文件模板

> 用途：定义生成 SKILL 的 scripts/ 下脚本文件的标准结构
> 核心原则：脚本优先只读，必须标注安全等级与副作用，禁止高危操作
> 设计方法详见 [design-guides/scripts-design-guide.md](../design-guides/scripts-design-guide.md)

---

## 1. 脚本头信息

```python
#!/usr/bin/env python3
"""
{脚本名称}

用途：{一句话说明用途}
安全等级：{安全 / 中风险 / 高风险}
输入：{输入文件 / 数据}
输出：{输出格式}
副作用：{无 / 读取 / 写入 / 网络}

作者：{生成者}
日期：{YYYY-MM-DD}
"""
```

---

## 2. 安全原则

| 原则 | 要求 |
|:---|:---|
| 只读优先 | 优先读取，不修改文件系统 |
| 幂等性 | 多次执行结果一致 |
| 自包含 | 不依赖外部环境 |
| 最小权限 | 只请求必要权限 |
| 输入验证 | 验证所有输入，拒绝非法输入 |

## 4. 脚本类型选择

| 类型 | 用途 | 安全等级 |
|:---|:---|:---:|
| 扫描脚本 | 静态检查，识别问题 | 安全 |
| 验证脚本 | 验证格式、约束、合规性 | 安全 |
| 生成脚本 | 生成内容、填充模板 | 中风险 |
| 转换脚本 | 格式转换、数据迁移 | 中风险 |
| 部署脚本 | 部署到生产环境 | 高风险 |

---

## 5. 完整性检查清单

- [ ] 脚本有安全等级标注
- [ ] 脚本有副作用说明
- [ ] 脚本有输入验证
- [ ] 脚本不执行高危操作（删除、写入、网络、系统命令）
- [ ] 脚本输出到 stdout，不直接写入文件
- [ ] 脚本有错误处理
- [ ] 脚本有日志记录
- [ ] 中风险及以上脚本经过人工审核
- [ ] 脚本自包含，不依赖外部环境
- [ ] 脚本幂等，多次执行结果一致
- [ ] 文件 < 200 行
