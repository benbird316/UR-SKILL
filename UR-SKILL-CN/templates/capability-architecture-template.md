# 能力架构模板

> 用途：定义 SKILL 的能力矩阵与能力切面的标准填写格式
> 核心原则：能力矩阵是能力领域，不是工作流步骤；能力切面只针对核心领域
> 设计方法详见 [design-guides/capability-design-guide.md](../design-guides/capability-design-guide.md)

---

## 1. 能力矩阵

### 1.1 核心领域

| 领域 | 基础层 | 进阶层 | 高阶层 | 拓展层 |
|:---|:---|:---|:---|:---|
| 核心：{核心领域名} | {核心能力} | {熟练运用} | {深度优化} | {上下文推理} |

### 1.2 辐射领域

| 领域 | 基础层 | 进阶层 | 高阶层 | 拓展层 |
|:---|:---|:---|:---|:---|
| A {领域1} | {核心能力} | {熟练运用} | {深度优化} | {上下文推理} |
| B {领域2} | {核心能力} | {熟练运用} | {深度优化} | {上下文推理} |
| C {领域3} | {核心能力} | {熟练运用} | {深度优化} | {上下文推理} |
| ... | ... | ... | ... | ... |

> 辐射领域数量由任务分析确定，推荐 3-8 个。详见 [design-guides/capability-design-guide.md §1.2](../design-guides/capability-design-guide.md)。

### 1.3 正例与反例

| 类型 | 示例 | 说明 |
|:---|:---|:---|
| 正例 | A 静态分析 / B 安全审计 / C 性能诊断 | 专业领域，重排后仍然成立 |
| 反例 | A 解析需求 / B 检索资料 / C 生成报告 | 工作流步骤，见 [design-guides/capability-design-guide.md §1.1](../design-guides/capability-design-guide.md) |

## 2. 能力切面

### 2.1 通用模板

| 编号 | 切面 | 定义 | 核心问题 |
|:---:|:---|:---|:---|
| 切面1 | 效率成本 | {该任务的具体资源约束与调节策略} | 资源投入是否合理？ |
| 切面2 | 知识深耕 | {该任务所需的具体知识栈——工具名、规范名、漏洞类型、标准号} | 知识覆盖是否完整？ |
| 切面3 | 风险识别 | {该任务中可能出现的具体反模式与风险场景} | 有什么潜在风险？ |
| 切面4 | 质量检验 | {该任务输出的具体验收标准} | 输出质量是否可靠？ |
| 切面5 | 领域融合 | {该任务的辐射领域之间如何互补无重叠} | 领域之间是否协同？ |
| 切面6 | 系统全局 | {该任务输出与上下游系统/平台的兼容性考虑} | 是否考虑了全局影响？ |

### 2.2 填写格式

```
能力切面（针对核心领域：{核心领域名}）

切面1 效率成本：{该任务的具体资源约束与调节策略}
切面2 知识深耕：{该任务所需的具体知识栈——工具名、规范名、漏洞类型、标准号}
切面3 风险识别：{该任务中可能出现的具体反模式与风险场景}
切面4 质量检验：{该任务输出的具体验收标准}
切面5 领域融合：{该任务的辐射领域之间如何互补无重叠}
切面6 系统全局：{该任务输出与上下游系统/平台的兼容性考虑}
```

> 任务锚定要求：每个切面必须包含该核心领域的具体知识，不能只写通用套话。详见 [design-guides/capability-design-guide.md §3](../design-guides/capability-design-guide.md)。

### 2.3 正例与反例

| 类型 | 示例 | 说明 |
|:---|:---|:---|
| 正例 | 切面2 知识深耕：掌握 Python 3.x 语法、PEP 8 规范、OWASP Python Top 10 | 任务锚定 |
| 反例 | 切面2 知识深耕：掌握相关领域知识 | 通用套话，见 [design-guides/capability-design-guide.md §6](../design-guides/capability-design-guide.md) |

## 3. 完整性检查清单

- [ ] 能力矩阵：核心领域 1 个，辐射领域 3-8 个，由任务分析确定
- [ ] 能力矩阵：每个领域 4 层完整（基础 → 进阶 → 高阶 → 拓展）
- [ ] 能力矩阵：领域是能力领域，不是工作流步骤
- [ ] 能力矩阵：核心领域与辐射领域之间独立且互补
- [ ] 能力切面：6 个切面，只针对核心领域，不重复建设到辐射领域
- [ ] 能力切面：每个切面有具体定义和核心问题
- [ ] 能力切面：每个切面任务锚定，不含通用套话
- [ ] 无占位符残留
- [ ] 能力矩阵不是工作流步骤的别名
- [ ] 能力切面不是能力领域的替代
