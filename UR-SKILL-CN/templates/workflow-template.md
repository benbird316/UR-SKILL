# 工作流模板

> 用途：定义 SKILL 工作流步骤的标准填写格式
> 核心原则：每步包含动作、检查清单、参考文件；动作绑定具体工具；检查清单基于审视维度
> 设计方法详见 [design-guides/tool-invocation-design-guide.md](../design-guides/tool-invocation-design-guide.md)
> 审视维度定义见 [design-rationale/design-rationale.md §4](../design-rationale/design-rationale.md)

---

## 1. 步骤结构模板

```markdown
#### {N}. {步骤名}【{节点类型}，{N} 维】

**动作**：
1. [{工具名}] {操作描述} → 声明 **{辐射领域}·{层级}**：{输出物}
2. [认知操作] {认知操作描述} → 激活 **{辐射领域}·{层级}**：{具体输出}
3. [{工具名}（↘ {降级工具}）] {操作描述} → 声明 **{辐射领域}·{层级}**：{输出物}

**核心命令**：确认...

**检查清单**：
- [ ] 目标对齐：...
- [ ] 事实锚定：...
- [ ] 方向校准：...（关键节点适用）
- [ ] 对抗验证：...（关键节点适用）
- [ ] 盲区识别：...
  - 盲区处理：（已尝试动作） / （剩余盲区） / （可行性建议）
- [ ] 影响推演：...（关键节点适用）
- [ ] 触及风险边界：（是/否） → 是 → 终止

→ 任一未确认 → 补齐 → 返回确认 → 全部确认 → 进入 {N+1}

**参考文件**：{references/...}
```

> 节点类型与审视维度：关键节点（调研、架构、校验、验证）6 维全开；非关键节点（解析、执行、交付）3 维（目标对齐、事实锚定、盲区识别）。详见 [design-rationale/design-rationale.md §4](../design-rationale/design-rationale.md)。

---

## 2. 动作格式

| 动作类型 | 格式 | 示例 |
|:---|:---|:---|
| 工具操作 | `[{工具名}] {操作} → {输出}` | `[Read] 读取需求输入 → 提取任务类型` |
| 认知操作 | `[认知操作] {描述} → 激活 **{领域}·{层级}**：{输出}` | `[认知操作] 基于Schema设计抽取Prompt → 激活 **B实体抽取·高阶层**：输出三元组` |
| 降级操作 | `[{工具名}（↘ {降级}）] {操作} → {输出}` | `[WebSearch（↘ WebFetch）] 搜索行业标准 → 获取领域知识` |

> 工具绑定规范详见 [design-guides/tool-invocation-design-guide.md](../design-guides/tool-invocation-design-guide.md)。

---

## 3. 复杂度与步骤数

| 复杂度 | 步骤数 |
|:---|:---:|
| 简单 | 3-5 步 |
| 中等 | 5-7 步 |
| 复杂 | 7 步以上 |

> 简单 vs 中等的区别是步骤数量，不是审视维度。

---

## 4. 盲区三层处理机制

盲区识别后必须按以下机制递进处理：

| 层级 | 动作 | 输出 |
|:---|:---|:---|
| 第一层 | 调查分析 → 自行优化填补 | 优化完成，返回确认 |
| 第二层 | 仍有不足 → 请求资源 | 资源补充后继续优化，返回确认 |
| 第三层 | 无资源补充 → 输出盲区处理报告 | 已尝试动作 + 剩余盲区 + 可行性建议，返回确认 |

> 禁止从第一层直接跳到第三层。详见 [design-rationale/design-rationale.md §7](../design-rationale/design-rationale.md)。

---

## 5. 填写规范

| 元素 | 规范 | 示例 |
|:---|:---|:---|
| 步骤名 | 动词开头，不超过 8 字 | 解析、调研、架构、执行、校验、验证、交付 |
| 动作 | 动词开头，有输出物；可执行动作必须绑定工具或标注认知操作，并声明能力域·层级 | `[Read] 读取用户需求 → 声明 **需求工程·基础层**：需求摘要` |
| 核心命令 | 命令句，"确认..." | 确认信息已足够支撑复杂度判定 |
| 参考文件 | 复数形式 references/ | design-rationale/design-rationale.md |

---

## 6. 完整性检查清单

- [ ] 每步包含动作、检查清单、参考文件三要素
- [ ] 每步骤只有 1 个核心命令
- [ ] 简单 SKILL 3-5 步，中等 5-7 步，复杂 7 步以上
- [ ] 审视维度按节点类型分配（关键节点 6 维，非关键节点 3 维）
- [ ] 可执行动作已绑定至少一个具体工具（格式：`[工具名] 操作 → 声明 **{领域}·{层级}**：输出`）或标注为 `[认知操作] → 激活 **{领域}·{层级}**：输出`
- [ ] 关键工具调用已标注降级路径
- [ ] 盲区识别后必须有具体行动（调查优化 / 请求资源 / 盲区报告 + 可行性建议）
- [ ] 未确认时执行补齐动作，禁止强行进入下一步
- [ ] 参考文件路径使用复数形式（references/）
