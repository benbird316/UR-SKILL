# 工作流模板

> 用途：定义 SKILL 工作流步骤的标准填写格式
> 核心原则：工作流由 4 个主节点组成，子节点按需选配。每步包含动作、检查清单、参考文件。
> 设计方法详见 [design-guides/workflow-design-guide.md](../design-guides/workflow-design-guide.md)

---

## 1. 工作流骨架

```markdown
## 工作流

### 分析

{按需选配：解析、协调、调研、规划}

### 执行

{必须含"执行"子节点，按需选配：分发、汇总}

### 反思

{必须含"决定"子节点，按需选配：校验、验证}

### 交付

{直接动作，无子节点}
```

---

## 2. 子步骤模板

每个子步骤（含交付）按以下格式填写：

```markdown
#### {N}. {步骤名}【{门控/执行}，{N}维】

**动作**：
1. [{工具名}] {操作描述} → 声明 **{辐射领域}·{层级}**：{输出物}

**核心命令**：确认{检查项}

**检查清单**：
- [ ] 目标对齐：{该步骤的目标是否达成}
- [ ] 事实锚定：{引用的信息是否真实可追溯}
{门控节点追加以下4项：}
- [ ] 方向校准：{方案是否符合初始目标}
- [ ] 对抗验证：{是否存在反例或边界情况}
- [ ] 影响推演：{当前决策对后续步骤的连锁影响}
- [ ] 盲区识别：...
  - 盲区处理：（已尝试动作） / （剩余盲区） / （可行性建议）

→ 任一未确认 → 补齐 → 返回确认 → 全部确认 → 进入 {N+1}

**参考文件**：{references/...}
```

---

## 3. 填写规范

| 元素 | 规范 | 示例 |
|:---|:---|:---|
| 步骤名 | 动词开头，不超过 4 字 | 解析、调研、规划、执行、校验、验证、决定 |
| 动作 | 动词开头，有输出物；可执行动作绑定工具或标注 `[认知操作]`，声明领域·层级 | `[文件读取] 读取需求输入 → 声明 **需求工程·基础层**：需求摘要` |
| 核心命令 | 命令句，"确认..." | 确认信息已足够支撑方案设计 |
| 检查清单 | 门控节点 6 项全填，执行节点只填目标对齐、事实锚定、盲区识别 3 项 | — |
| 参考文件 | 复数形式 `references/` | references/verification-patterns.md |

---

## 4. 动作格式

| 动作类型 | 格式 | 示例 |
|:---|:---|:---|
| 工具操作 | `[{工具名}] {操作} → {输出}` | `[文件读取] 读取需求输入 → 提取任务类型` |
| 认知操作 | `[认知操作] {描述} → 激活 **{领域}·{层级}**：{输出}` | `[认知操作] 基于Schema抽取Prompt → 激活 **实体抽取·高阶层**：三元组` |
| 降级操作 | `[{工具名}（↘ {降级}）] {操作} → {输出}` | `[WebSearch（↘ WebFetch）] 搜索行业标准 → 获取领域知识` |

## 5. 实例

### 5.1 全开实例（Meta-SKILL）

```markdown
## 工作流

### 分析

#### 1. 解析（需求分析）【执行，3维】
**动作**：
1. [文件读取] 读取用户输入 → 声明 **需求工程·基础层**：任务摘要（类型/领域/交付形式）
**核心命令**：确认任务类型和领域已明确
**检查清单**：
- [ ] 目标对齐：提取了任务类型、领域、交付形式
- [ ] 事实锚定：输入信息真实可追溯
- [ ] 盲区识别：无歧义，无需补充
→ 全部确认 → 进入 2
**参考文件**：—

#### 2. 协调（任务分解）【执行，3维】
...

#### 3. 调研（领域知识）【门控，6维】
...

#### 4. 规划（方案设计）【门控，6维】
...

### 执行

#### 5. 分发（委托子Agent）【执行，3维】
...

#### 6. 执行（核心产出）【执行，3维】
...

#### 7. 汇总（产物合并）【执行，3维】
...

### 反思

#### 8. 校验（质量检查）【门控，6维】
...

#### 9. 验证（对抗测试）【门控，6维】
...

#### 10. 决定（门控判定）【门控，6维】
**动作**：
1. [认知操作] 逐项核对校验与验证结果 → 激活 **质量工程·高阶层**：通过/不通过判决
**核心命令**：确认所有门控条件已满足
**检查清单**：
- [ ] 目标对齐：产出与初始需求一致
- [ ] 事实锚定：引用来源可追溯
- [ ] 方向校准：方案无偏离
- [ ] 对抗验证：边界情况已覆盖
- [ ] 盲区识别：已声明盲区及处理结果
- [ ] 影响推演：交付后无遗留风险
→ 通过 → 进入交付 / 不通过 → 退回修复
**参考文件**：references/anti-patterns.md

### 交付

#### 11. 交付（最终输出）【执行，3维】
**动作**：
1. [文件写入] 按输出模板写入最终文件 → 声明 **输出工程·基础层**：产物文件
**核心命令**：确认输出完整且格式正确
**检查清单**：
- [ ] 目标对齐：输出结构符合输出规范
- [ ] 事实锚定：内容基于前期确认信息
- [ ] 盲区识别：无遗漏
→ 完成交付
**参考文件**：references/output-spec.md
```

### 5.2 精简实例（对话引导型 SKILL）

```markdown
## 工作流

### 分析

#### 1. 解析意图【执行，3维】
**动作**：
1. [认知操作] 分析用户输入 → 激活 **对话工程·基础层**：意图分类
**核心命令**：确认用户意图明确
**检查清单**：
- [ ] 目标对齐：意图分类正确
- [ ] 事实锚定：基于实际输入
- [ ] 盲区识别：无歧义
→ 全部确认 → 进入 2
**参考文件**：—

### 执行

#### 2. 生成回复【执行，3维】
**动作**：
1. [认知操作] 根据意图类型生成回复 → 激活 **对话工程·进阶层**：回复草稿
**核心命令**：确认回复符合意图
**检查清单**：
- [ ] 目标对齐：回复匹配意图
- [ ] 事实锚定：内容有依据
- [ ] 盲区识别：无不安全内容
→ 全部确认 → 进入 3
**参考文件**：—

### 反思

#### 3. 决定（安全复核）【门控，6维】
**动作**：
1. [认知操作] 扫描回复内容 → 激活 **安全工程·基础层**：安全判定
**核心命令**：确认无不安全内容
**检查清单**：
- [ ] 目标对齐：意图未被曲解
- [ ] 事实锚定：引用准确
- [ ] 方向校准：回复未偏航
- [ ] 对抗验证：恶意输入已拦截
- [ ] 盲区识别：无遗漏风险
- [ ] 影响推演：输出无安全隐患
→ 通过 → 交付 / 不通过 → 退回重生成
**参考文件**：references/content-safety.md

### 交付

#### 4. 输出回复【执行，3维】
**动作**：
1. [认知操作] 输出最终回复 → 激活 **对话工程·基础层**：用户可见回复
**核心命令**：确认输出完整
**检查清单**：
- [ ] 目标对齐：输出即最终回复
- [ ] 事实锚定：内容已校验
- [ ] 盲区识别：无遗漏
→ 完成交付
**参考文件**：—
```

## 6. 完整性检查清单

- [ ] 4 个主节点完整（分析、执行、反思、交付）
- [ ] 执行主节点含"执行"子节点（必须）
- [ ] 反思主节点含"决定"子节点（必须）
- [ ] 其余子节点按需选配（参考 [workflow-design-guide.md §2](../design-guides/workflow-design-guide.md#2-子节点按需选配) 条件列逐项判定）
- [ ] 门控节点（调研、规划、校验、验证、决定）6 维全开
- [ ] 执行节点（解析、协调、分发、执行、汇总、交付）3 维
- [ ] 每步有动作、核心命令、检查清单、参考文件
- [ ] 每步只有 1 个核心命令
- [ ] 可执行动作已绑定工具或标注认知操作
- [ ] 门控节点盲区项含三层处理动作
- [ ] 参考文件路径使用复数形式
